## ⚡ **Quick Deploy (5 minutes)**

### **Railway Deployment**

**Step 1: Install Railway CLI**
```bash
npm install -g @railway/cli
```

**Step 2: Login to Railway**
```bash
railway login
```

**Step 3: Initialize Project**
```bash
cd debugg/dashboard
railway init
```

**Step 4: Add PostgreSQL**
```bash
railway add postgresql
```

**Step 5: Add Redis**
```bash
railway add redis
```

**Step 6: Set Environment Variables**
```bash
railway variables set NODE_ENV=production
railway variables set DATABASE_URL=<auto-generated>
railway variables set REDIS_URL=<auto-generated>
railway variables set DASHBOARD_API_KEY=<your-secure-key>
```

**Step 7: Deploy**
```bash
railway up
```

**Done!** Your dashboard is live at `https://your-project.railway.app`

---

### **Render Deployment**

**Step 1: Create render.yaml**
```yaml
services:
  - type: web
    name: debugg-dashboard
    env: node
    region: oregon
    plan: starter
    buildCommand: npm install && npx prisma generate
    startCommand: node dist/server.js
    envVars:
      - key: NODE_ENV
        value: production
      - key: DATABASE_URL
        fromDatabase:
          name: debugg-db
          property: connectionString
      - key: REDIS_URL
        fromDatabase:
          name: debugg-redis
          property: connectionString
      - key: DASHBOARD_API_KEY
        generateValue: true

databases:
  - name: debugg-db
    databaseName: debugg
    plan: starter
    
  - name: debugg-redis
    plan: starter
```

**Step 2: Push to GitHub**
```bash
git add .
git commit -m "Initial commit"
git push origin main
```

**Step 3: Connect in Render**
1. Go to https://render.com
2. Click "New +" → "Blueprint"
3. Connect your GitHub repository
4. Deploy!

## 🏭 **Production Deploy (30 minutes)**

### **DigitalOcean Deployment**

**Step 1: Create Droplet**
```bash
# Via DO CLI
doctl compute droplet create debugg-dashboard \
  --region nyc3 \
  --image ubuntu-22-04-x64 \
  --size s-2vcpu-4gb \
  --ssh-keys your-key-fingerprint
```

**Step 2: Connect to Server**
```bash
ssh root@your-server-ip
```

**Step 3: Install Dependencies**
```bash
# Update system
apt update && apt upgrade -y

# Install Node.js
curl -fsSL https://deb.nodesource.com/setup_20.x | bash -
apt install -y nodejs

# Install PostgreSQL
apt install -y postgresql postgresql-contrib

# Install Redis
apt install -y redis-server

# Install Nginx
apt install -y nginx

# Install PM2
npm install -g pm2
```

**Step 4: Setup PostgreSQL**
```bash
sudo -u postgres psql

CREATE DATABASE debugg;
CREATE USER debugg_user WITH PASSWORD 'secure-password';
GRANT ALL PRIVILEGES ON DATABASE debugg TO debugg_user;
\q
```

**Step 5: Clone Repository**
```bash
cd /var/www
git clone https://github.com/your-org/debugg.git
cd debugg/dashboard
```

**Step 6: Install Dependencies**
```bash
npm install
npx prisma generate
npx prisma migrate deploy
npm run build
```

**Step 7: Configure Environment**
```bash
nano .env

# Add:
NODE_ENV=production
DATABASE_URL=postgresql://debugg_user:secure-password@localhost:5432/debugg
REDIS_URL=redis://localhost:6379
DASHBOARD_API_KEY=your-secure-api-key
PORT=3001
```

**Step 8: Setup PM2**
```bash
pm2 start dist/server.js --name debugg-dashboard
pm2 save
pm2 startup
```

**Step 9: Configure Nginx**
```bash
nano /etc/nginx/sites-available/debugg-dashboard

# Add:
server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://localhost:3001;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
    }
}
```

**Step 10: Enable Site**
```bash
ln -s /etc/nginx/sites-available/debugg-dashboard /etc/nginx/sites-enabled/
nginx -t
systemctl reload nginx
```

**Done!** Your dashboard is live at `http://your-domain.com`

## 🔒 **Enterprise Deploy (1 hour)**

### **AWS Deployment with Docker**

**Step 1: Create Dockerfile**
```dockerfile
FROM node:20-alpine

WORKDIR /app

# Install dependencies
COPY package*.json ./
RUN npm ci --production

# Copy application
COPY . .

# Generate Prisma client
RUN npx prisma generate
RUN npx prisma migrate deploy

# Build application
RUN npm run build

# Expose port
EXPOSE 3001

# Health check
HEALTHCHECK --interval=30s --timeout=3s \
  CMD wget -q --spider http://localhost:3001/health || exit 1

# Start application
CMD ["node", "dist/server.js"]
```

**Step 2: Create docker-compose.yml**
```yaml
version: '3.8'

services:
  app:
    build: .
    ports:
      - "3001:3001"
    environment:
      - NODE_ENV=production
      - DATABASE_URL=postgresql://debugg:secure@db:5432/debugg
      - REDIS_URL=redis://redis:6379
      - DASHBOARD_API_KEY=${DASHBOARD_API_KEY}
    depends_on:
      - db
      - redis
    restart: unless-stopped

  db:
    image: postgres:15-alpine
    environment:
      - POSTGRES_USER=debugg
      - POSTGRES_PASSWORD=secure
      - POSTGRES_DB=debugg
    volumes:
      - postgres_data:/var/lib/postgresql/data
    restart: unless-stopped

  redis:
    image: redis:7-alpine
    volumes:
      - redis_data:/data
    restart: unless-stopped

volumes:
  postgres_data:
  redis_data:
```

**Step 3: Deploy to AWS ECS**
```bash
# Install AWS CLI
npm install -g aws-cli

# Login to ECR
aws ecr get-login-password --region us-east-1 | \
  docker login --username AWS --password-stdin <account>.dkr.ecr.us-east-1.amazonaws.com

# Build and push image
docker build -t debugg-dashboard .
docker tag debugg-dashboard:latest <account>.dkr.ecr.us-east-1.amazonaws.com/debugg-dashboard:latest
docker push <account>.dkr.ecr.us-east-1.amazonaws.com/debugg-dashboard:latest

# Deploy to ECS
aws ecs create-cluster --cluster-name debugg-cluster
aws ecs create-service --cluster debugg-cluster --service-name debugg-service \
  --task-definition debugg-task --desired-count 2
```

## 🔧 **Monitoring & Maintenance**

### **PM2 Commands**
```bash
# View logs
pm2 logs debugg-dashboard

# Restart app
pm2 restart debugg-dashboard

# View status
pm2 status

# View memory
pm2 monit
```

### **Database Backup**
```bash
# Daily backup script
#!/bin/bash
pg_dump -U debugg debugg > /backups/debugg-$(date +%Y%m%d).sql

# Add to crontab
0 2 * * * /path/to/backup.sh
```

### **Log Rotation**
```bash
# /etc/logrotate.d/debugg-dashboard
/var/log/debugg-dashboard/*.log {
    daily
    rotate 14
    compress
    delaycompress
    notifempty
    create 0640 root root
}
```

### **Health Check Script**
```bash
#!/bin/bash

# Check if app is running
if ! pm2 list | grep -q "debugg-dashboard"; then
  echo "App is not running! Restarting..."
  pm2 restart debugg-dashboard
fi

# Check database
if ! pg_isready -U debugg; then
  echo "Database is not ready!"
fi

# Check Redis
if ! redis-cli ping; then
  echo "Redis is not ready!"
fi
```

## 📞 **Troubleshooting**

### **App Won't Start**
```bash
# Check logs
pm2 logs debugg-dashboard --lines 100

# Check environment
pm2 show debugg-dashboard

# Check database connection
psql postgresql://debugg_user:password@localhost:5432/debugg
```

### **Database Connection Failed**
```bash
# Check PostgreSQL status
systemctl status postgresql

# Check connection
sudo -u postgres psql -c "SELECT 1"

# Restart PostgreSQL
systemctl restart postgresql
```

### **Redis Connection Failed**
```bash
# Check Redis status
systemctl status redis

# Test connection
redis-cli ping

# Restart Redis
systemctl restart redis
```

### **High Memory Usage**
```bash
# Check memory
pm2 monit

# Restart app
pm2 restart debugg-dashboard

# Check for memory leaks
pm2 logs debugg-dashboard | grep -i "memory"
```
