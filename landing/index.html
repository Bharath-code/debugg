Questions.

What exactly is Debugg?

A comprehensive, cross-platform error handling and monitoring library. It provides unified error handling with automatic severity classification and multiple reporters with an ultra-minimal footprint.

Is there a performance overhead?

Almost none. Debugg is optimized for production. Error creation takes < 0.1ms, handling takes < 1ms, and it consumes < 1KB of memory per error. It is built for pure efficiency.

Why choose Debugg over Sentry?

We offer automatic classification out-of-the-box, built-in rate limiting, and an incredibly lightweight bundle size (~73 KB). Most importantly, we're 10x cheaper for comparable scale.

Does it support Next.js or React Native?

Yes. Debugg has cross-platform support. Whether you use Node, Edge, traditional React, or React Native, you integrate once with a unified API.
