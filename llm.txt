## 🎯 **Key API Endpoints**

### **Base URL**
```
http://localhost:3001/api
```

### **Authentication**
```http
X-API-Key: your-api-key
```

### **Endpoints**

#### **Errors**
```
GET    /api/errors              # List errors
GET    /api/errors/:id          # Get error by ID
POST   /api/errors              # Create error
PATCH  /api/errors/:id/status   # Update error status
DELETE /api/errors/:id          # Delete error
```

#### **Users**
```
GET    /api/users               # List users
POST   /api/users               # Create user
PATCH  /api/users/:id           # Update user
DELETE /api/users/:id           # Delete user
```

```
GET    /api/audit-logs          # List audit logs
GET    /api/audit-logs/export   # Export CSV
```

```
GET    /api/analytics/overview  # Analytics overview
GET    /api/analytics/trends    # Error trends

```

## 🎨 **Usage Patterns**

### **Pattern 1: Basic Error Handling**
```typescript
import { debugg } from 'debugg';

try {
  await riskyOperation();
} catch (error) {
  await debugg.handle(error, {
    userId: '123',
    action: 'checkout',
  });
}
```

### **Pattern 2: Custom Context**
```typescript
await debugg.handle(error, {
  // User context
  userId: user.id,
  userEmail: user.email,
  
  // Request context
  endpoint: req.path,
  method: req.method,
  headers: req.headers,
  
  // Business context
  action: 'create_order',
  amount: 99.99,
  currency: 'USD',
});
```

### **Pattern 3: Custom Reporter**
```typescript
import { ErrorReporter } from 'debugg';

const customReporter: ErrorReporter = async (error) => {
  await fetch('https://your-service.com/errors', {
    method: 'POST',
    body: JSON.stringify(error),
  });
};

debugg.addReporter(customReporter);
```

### **Pattern 4: Error Boundary (React)**
```typescript
import { debugg } from 'debugg';

class ErrorBoundary extends React.Component {
  componentDidCatch(error, errorInfo) {
    debugg.handle(error, {
      component: this.constructor.name,
      ...errorInfo,
    });
  }
  
  render() {
    return this.props.children;
  }
}
```

## 📊 **Error Severity Levels**

```typescript
type ErrorSeverity = 
  | 'critical'   // System crashes, syntax errors
  | 'high'       // Type errors, network failures
  | 'medium'     // Standard errors (default)
  | 'low'        // Range errors, minor issues
  | 'info';      // Informational messages
```

**Automatic Classification:**
- `SyntaxError` → critical
- `TypeError` → high
- `ReferenceError` → high
- `NetworkError` → high
- `RangeError` → medium
- Default → medium

## 🤖 **LLM-Specific Instructions**

### **When Generating Code with Debugg:**

1. **Always import correctly**
   ```typescript
   import { debugg } from 'debugg';
   ```

2. **Use handle() for errors**
   ```typescript
   await debugg.handle(error, { context });
   ```

3. **Include relevant context**
   - User information
   - Request details
   - Business context
   - Technical details

4. **Configure for environment**
   ```typescript
   const debuggInstance = new debugg.EnhancedErrorHandler({
     serviceName: 'my-app',
     environment: process.env.NODE_ENV,
   });
   ```

5. **Add appropriate reporters**
   ```typescript
   debugg.addReporter(debuggInstance.createConsoleReporter());
   ```

### **When Explaining Debugg:**

1. **Highlight key benefits**

   - Enterprise-ready

   - Basic usage

   - Advanced patterns

3. **Mention integrations**
   - React, Vue, Angular
   - Express, Next.js, NestJS
   - React Native
   - Custom reporters

4. **Reference documentation**
   - Point to specific docs
